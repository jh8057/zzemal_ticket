# 티켓팅 연습 페이지 — 요구사항

## 서비스 개념

매 5분마다 열리는 티켓팅 대기열을 실제와 동일한 흐름으로 반복 연습하는 앱.
하루 기준으로 회차가 누적되며, 1분 단위의 빠른 연습 모드도 제공한다.

---

## 모드 정의

### 모드 1: 실제 서비스 테스트 (5분 단위)
- 매 :00, :05, :10... 정각마다 새 회차 오픈
- 하루 자정(00:00)부터 5분마다 1회차 증가 (오늘 121회차, 122회차 등)
- 플로우: **대기(standby) → 대기열 1분 → 좌석 선택 4분 → 결과**
  - standby: 다음 5분 경계까지 카운트다운. 경계 도달 시 1분 window 내에 "대기열 입장" 버튼 활성화
  - queue: 가짜 대기번호 카운트다운 1분 (자동 전이)
  - selecting: 4분 안에 좌석 선택
  - confirming: 선택 좌석 확인 후 예매 확정
  - result: 소요 시간 표시 + 다음 회차 도전

### 모드 2: 대기열 빠른 연습 (1분 단위)
- 매 1분 정각마다 자동 시작
- 플로우: **대기(standby) → 좌석 선택 45초 → 결과**
  - standby: 다음 1분 경계까지 카운트다운 (SS 표시), 자동 전이
  - selecting: 45초 안에 좌석 선택
  - confirming: 선택 확인
  - result: 소요 시간 + 점수 + 다시 연습

## 기능 요구사항

### Must Have
| ID | 기능 | 설명 |
|----|------|------|
| F-01 | 모드 선택 | 홈에서 실서비스/빠른연습 모드 선택 |
| F-02 | 회차 표시 | 오늘 기준 5분 단위 회차 번호 표시 (실서비스 모드) |
| F-03 | 카운트다운 | 다음 오픈까지 MM:SS 실시간 표시 |
| F-04 | 대기열 시뮬레이션 | 가짜 대기번호 카운트다운으로 실제 대기 경험 제공 |
| F-05 | 좌석 선택 | 구역/행/번호 시각화, available/held/booked 상태 표시 |
| F-06 | 예매 확정 | 좌석 선택 후 확정 버튼 → 결과 표시 |
| F-07 | 소요 시간 측정 | 좌석 선택 시작~예매 완료까지 초 단위 측정 |
| F-08 | 타임아웃 | 제한 시간 초과 시 자동으로 결과 페이지 전환 |

### Nice to Have
| ID | 기능 | 설명 |
|----|------|------|
| F-10 | 기록 저장 | 회차별 소요 시간 로컬 저장 및 히스토리 표시 |
| F-11 | 실제 대기열 | 백엔드 연동 후 실제 동시 접속자 처리 |
| F-12 | 동시성 테스트 | 같은 좌석에 동시 요청 시 하나만 성공하는 시나리오 |

## 비기능 요구사항
- 회차 계산은 로컬 시간 기준 (한국 사용자 대상)
- 새로고침 시 standby 단계로 초기화 (Phase 1)
- 백엔드 없이 프론트 단독으로 완전히 동작해야 함 (Phase 1)
