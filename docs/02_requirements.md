# 티켓팅 연습 페이지 — 요구사항

## 기능 요구사항

### Must Have
| ID | 기능 | 설명 |
|----|------|------|
| F-01 | 공연 목록 조회 | 공연명, 날짜, 잔여 좌석 표시 |
| F-02 | 좌석 선택 | 구역/행/번호 시각화, 선택/예약됨/빈자리 상태 표시 |
| F-03 | 예매 요청 | 좌석 선택 후 예매 버튼 → 임시 선점 (N분 유효) |
| F-04 | 예매 확정 | 선점 상태에서 "예매하기" 버튼 → 예매 완료 처리 |
| F-05 | 예매 완료 확인 | 예매 내역 조회 (티켓 번호, 좌석, 공연 정보) |

### Nice to Have
| ID | 기능 | 설명 |
|----|------|------|
| F-10 | 대기열 | 동시 접속자 초과 시 대기 번호 부여 및 대기열 UI |
| F-11 | 타이머 | 좌석 임시 선점 후 예매 제한 시간 카운트다운 |
| F-12 | 동시성 테스트 | 같은 좌석에 동시 요청 시 하나만 성공하는지 확인용 UI |

## 비기능 요구사항
- **성능**: 동시에 같은 좌석에 여러 요청이 와도 중복 예매 없어야 함
- **보안**: 임시 선점된 좌석은 다른 사용자가 선택 불가
- **확장성**: 공연/좌석 데이터는 DB에서 관리 (하드코딩 X)
- **기타**: 새로고침해도 선점 상태 유지 (서버 사이드 상태)
