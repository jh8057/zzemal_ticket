# 티켓팅 연습 페이지 — 완료 조건

## Phase 1 완료 조건 (프론트 단독)

### 메인 페이지
- [ ] 1분 단위 카운트다운 정확히 동작 (매 :00초에 리셋)
- [ ] "혼자 연습하기" 버튼 → /practice 이동
- [ ] "대기열 참여" 버튼 → 비활성 또는 "준비 중" 표시 (Phase 2 전까지)

### 혼자 연습 페이지
- [ ] 좌석 배치도 렌더링 (일부 무작위 선점 상태)
- [ ] 좌석 클릭 → 타이머 시작
- [ ] "예약하기" 버튼 클릭 → 타이머 중지 + 경과 시간 표시
- [ ] 속도 등급 표시 (<2s 번개, <5s 빠름, <10s 평균, 이상 느림)
- [ ] "다시 시도" 버튼

### S3 배포
- [ ] `npm run build` 정상 완료
- [ ] S3 버킷 정적 웹사이트 호스팅 설정 완료
- [ ] 도메인(또는 S3 URL)으로 접속 가능

## Phase 2 완료 조건 (Lambda + DynamoDB)

### 좌석 선점
- [ ] 빈 좌석 클릭 → DynamoDB 조건부 쓰기로 선점 성공
- [ ] 이미 선점된 좌석 클릭 → "이미 선택된 좌석" 안내
- [ ] 5분 후 DynamoDB TTL로 자동 선점 해제

### 예매 확정
- [ ] "예매 확정" 버튼 → DB에 Booking 저장
- [ ] 예약된 좌석은 영구적으로 선택 불가
- [ ] 티켓 번호 발급 + 예매 완료 페이지 이동

### 동시성
- [ ] 같은 좌석에 동시 요청 시 하나만 성공 (DynamoDB ConditionExpression)

## 주요 테스트 시나리오

| # | 시나리오 | 예상 결과 |
|---|---------|----------|
| 1 | 빈 좌석 클릭 → 예매 확정 | 예매 성공, 좌석 상태 "예약됨" |
| 2 | 이미 선점된 좌석 클릭 | "이미 선택된 좌석" 안내 |
| 3 | 선점 후 5분 대기 | 선점 자동 해제, 다시 선택 가능 |
| 4 | 같은 좌석에 동시 요청 2개 | 1개만 성공, 1개는 409 |
| 5 | 혼자 연습 페이지 — 좌석 클릭 후 예약하기 | 경과 시간 + 등급 표시 |
