# 티켓팅 연습 페이지 — 프로젝트 개요

## 한 줄 설명

> 실제 티켓팅 대기열 흐름을 반복 연습할 수 있는 웹 애플리케이션

## 배경 & 목적

- 실제 콘서트 티켓팅에서 겪는 대기열 진입 타이밍, 좌석 선택 속도를 사전에 연습
- 5분 단위로 회차를 부여해 실제 티켓팅 오픈 상황과 유사한 환경 제공
- 반응속도, 좌석 선택 속도를 수치로 측정해 개선 여부를 확인할 수 있도록 구현
- 실제와 완전 유사한 환경보다 핵심 흐름(대기열 → 좌석 선택 → 예매 확정)에 집중
- 백엔드 없이 프론트 단독으로 먼저 완성, 이후 실제 대기열(Redis) 연동 예정

## 4가지 연습 모드

| 모드 | 주기 | 설명 |
|------|------|------|
| 실서비스 테스트 | 5분 단위 | 하루 기준 회차 부여. 대기열 1분 → 좌석 선택 4분 |
| 대기열 빠른 연습 | 1분 단위 | 1분마다 버튼 오픈. 대기열 15초 → 좌석 선택 45초 |
| 대기열 반응속도 연습 | 10초 간격 | 버튼 활성화 시 클릭 반응속도(ms) 측정 |
| 좌석 선택 속도 연습 | 자유 시작 | 좌석 클릭~예매 확정 소요 시간 측정, 구간별 분리 |

## 핵심 목표

- [x] 1분 단위 빠른 연습 — 버튼 클릭으로 대기열 진입
- [x] 반응속도/좌석 속도 개인 연습 모드
- [ ] 실제 대기열 (Redis Queue 기반, Phase 3)
- [ ] 동시 접속자와 경쟁하는 멀티유저 환경 (Phase 3)

## 대상 사용자

티켓팅 경쟁에서 이기고 싶은 콘서트 팬, 티켓팅 시스템 구조를 학습하는 개발자

## 범위

**In Scope (구현 완료 또는 예정)**

- 4가지 연습 모드 (Phase 1 완료)
- 좌석 선택 UI (구역/행/번호 시각화)
- 소요 시간 측정 및 결과 표시
- 대기열 시뮬레이션 (Phase 2: 실제 Redis 연동)
- 실시간 좌석 경쟁 (Phase 3)

**Out of Scope**

- 실제 PG 연동
- 회원 관리 시스템
- 모바일 앱
