# 티켓팅 연습 페이지 — 프로젝트 개요

## 한 줄 설명

> 콘서트/공연 티켓팅 상황을 직접 경험해볼 수 있는 연습용 웹 애플리케이션

## 배경 & 목적

<!-- 왜 만드는가? 어떤 문제를 해결하는가? -->

- 실제 티켓팅 경쟁 환경(동시 접속, 좌석 선점, 결제 흐름)을 체험해볼 수 있는 페이지
- 티켓팅 시스템의 핵심 기술(동시성 처리, 대기열, 재고 관리 등) 학습
- 실제 도메인으로 접속 가능하도록해서 실제 티켓팅 시스템과 유사한 환경에서 연습할 수 있도록 구현
- 실제와 완전 유사한 환경 구축보단 핵심 기능에 집중하여 빠르게 구현하는 것을 목표로 함
- 메인페이지는 1분단위로 대기열 시뮬레이션이 가능한 버튼과 좌석 선택 페이지로 이동하는 버튼 2개로 구성
- 대기열 시뮬시, 가상의 300명 사용자가 함께 경쟁함
- 가상의 300명의 사용자는 정해진 시간에 버튼 클릭을 한다고 가정함 (ex. 0.3초에 10명, 0.6초에 50명 등)
- 대기열 시뮬레이션이 끝나면 좌석 선택 페이지로 이동하여 실제 티켓팅과 유사한 환경에서 연습할 수 있도록 구현
- 이때 대기열을 실제 티켓팅 시스템에서처럼 Redis Queue로 구현하여 대기열이 끝난 후 좌석 선택 페이지로 이동하도록 구현
- 가상의 300명은 대기열에 미포함
- 실제 접속한 다른 사용자와 경쟁하는 환경에서 연습할 수 있도록 구현
- 좌석 페이지는 총 2개야. 혼자서 하는 연습용 페이지와 대기열 시뮬레이션이 끝난 후 접속하는 페이지.
- 혼자서 연습하는 페이지는 단순하게 랜덤하게 남은 좌석이 보이고, 클릭 후 예약하기 버튼을 누를때까지 시간을 측정하는 형태로 구현
- 대기열 시뮬레이션이 끝난 후 접속하는 페이지는 실제 티켓팅과 유사한 환경에서 연습할 수 있도록 구현.
- 메인페이지는 최대한 단순하게 들어오자마자 아 1분 단위로 티켓팅을 할 수 있구나 를 느낄수 있게 구현.

## 대상 사용자

티켓팅 시스템 구조를 이해하고 싶은 콘서트 팬, 개발자, 학생 등

## 범위

**In Scope (만들 것)**

- 공연 목록 페이지
- 좌석 선택 페이지
- 대기열 시뮬레이션
- 예매 확정 (선점 → 예매 완료)

**Out of Scope (만들지 않을 것)**

- 실제 PG 연동
- 회원 관리 시스템 (간단한 로그인만)
- 모바일 앱
